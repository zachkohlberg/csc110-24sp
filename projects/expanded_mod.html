CSC110 Projects

Modified Expanded Projects

If you don't find one of the expanded projects very interesting and would prefer to code something else, then I encourage you to propose an alternative project. This is more work, but the extra effort can be worth it if you're spending your time on something that you enjoy more than the default project. This page describes how all of the requirements for a modified/alternative expanded project. You can also use this to propose an alternative to guided projects.

Learning Objectives

Pay attention to the learning objectives in the project you are modifying/replacing. Some of these will be covered by the fact that you are designing and proposing your own project, but you should still check to make sure your proposal addresses each learning objective. I'm not likely to approve a proposal that doesn't cover the same objectives as the assignment it's replacing.

Summary

Your proposal must summarize your project. You don't need to get into detail here; I just want on overview of the program you plan to write so that I know what to expect. I recommend sending me this summary before you go any further with the proposal. If I have concerns about an idea or don't think it will work, then I'd rather tell you this before you time writing the rest of the proposal.

Tasks

Explain each task you will complete. This is where you should go into detail. I expect something comparable to the task explanations in my assignments. When I write directions for a task, I'm trying to make it clear what I expect you to do and provide a little guidance. There are three reasons I need some detail here:

  • I need to determine whether this meets the learning objectives for the project it will replace.
  • I need to determine whether your project is an appropriate scope and difficulty. I don't want you to take on something that will be too large or challenging to finish, and I don't want you using the alternate projects to trivialize assignments that are meant to help you improve as a programmer.
  • I need to determine whether the work you turned in is complete.

Write-Up

You must still complete a write-up for your project, but it will only include the two standard questions I put on every open-ended project write-up. You've already done a fair amount of writing in creating your proposal, so that is an appropriate substitute for the assignment-specific questions.

The write-up for your modified assignment must address the following questions:

  1. Review your program and find a piece of code that was either challenging to write or that you think is an example of particularly good code (it's up to you what "good code" means, but you should be able to justify your opinion). Explain why you chose this code: why is it an example of good code or what was challenging about writing it? Include the code in your answer and note the line number(s) where it appears in your program.
  2. Review your program and find a piece of code that you think could be improved. What do you dislike or want to change about this code? If you were to spend more time on the project, how might you improve this code? To be clear, I'm not asking you how the program could be better. I don't want to know what feature you'd add or how you'd make it easier to use. I'm asking how part of the code could be better. This might mean shorter, easier to read, less repetitive, or something else that you value and think you could improve with additional time and/or programming knowledge. Include the code in your answer and note the line number(s) where it appears in your program.

I otherwise have the same expectations for this write-up as any other project write-up. Address each question in full and don't use an "AI" program to assist you.

Deliverables

Your proposal should state any deliverables aside from the write-up. This will probably just be your program, but depending on the project you may decide to include other deliverables, such as test data or a README.

In order for an open-ended project to be considered complete, it must

  • be submitted on Blackboard before the final deadline
  • compile and run without syntax errors
  • demonstrate that you have completed each task as directed
  • include a document that fully addresses each of the write-up questions

If you ignore write-up questions, tasks, or the directions within a task, then the project is incomplete. You can still resubmit an incomplete project, but resubmitting after the deadline will incur a significant late penalty.

Deadline

If you are replacing an expanded project, then your project is still due on the same day as the project it replaced. If you think you'll need extra time, then read the deadline/extension policy in the syllabus.

If you are replacing one or more guided projects, then we'll need to discuss an appropriate deadline. I allow guided projects to be turned in up until the last week of classes because they don't take time for me to review. However, reviewing open-ended assignments is much more time consuming, and I need to make sure these don't all pile up at the end of the semester. By default, I'll want to use the same deadline as whatever expanded project normally follows the guided projects you are replacing.

Finally, whatever deadline you have for the modified project, you need to get the project approved at least one week before that deadline. I don't want to allow or encourage alternative projects as last-minute attempts to get around project requirements. This is something you need to start thinking about early and tell me about well before the assignment itself is due.

Proposal Revisions

You can revise your proposal up to a week before the deadline. Once we're down to a week, you're committed to the most recent version of your proposal that I approved. A revision would be appropriate if you underestimated the difficulty of a task or thought of a better idea for part of the project.
