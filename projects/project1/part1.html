CSC110 Projects

Guided Project 1.1

This program will prompt the user for a few pieces of information about their character, calculate some additional stats, and then print a character sheet.

Learning Objectives

  • Use I/O functions in a Jshell script to interact with the user
  • Use variables to store user input
  • Use variables in simple calculations
  • Use variables in a program's output

Prerequisites and Setup

  • Create a file named CharacterSheet1.jsh in your project1/part1 folder.

Tasks

Task 1: Guided Coding

Follow the steps below and write the program as directed. Test your code frequently, at the bare minimum performing every test given in the steps.

If your code is not working as expected, do not move on to the next step until you have fixed it. Moving on without finishing a step correctly will make the problem significantly harder to fix and likely add new problems as you continue working. If you disregard this advice, then you may get to the end of the steps and find it would actually be less work to start over from step 1 than to debug the code you just finished writing.

  1. I recommend using comments to divide your program into sections so that it's easier to stay organized. I'll be referring to three different parts of the program in these steps: input, calculations, and output. Before the input section, add comments with a title for the program and the author's name (that's you). Feel free to copy the template below and fill in the missing information.
// Project 1.1: Character Sheet
// Author: your name here

///////////
// Input //
///////////

// TODO: prompt for user input

//////////////////
// Calculations //
//////////////////

// TODO: calculations

////////////
// Output //
////////////

// TODO: print character sheet
  1. In the input section, prompt the user for the following information in order: their name, their character's name, their character's level, and their character's ability scores (strength, dexterity, and constitution). When you prompt the user for input, remember that this involves two lines of code:
    • A println to ask the user for input. You want the user to know that they're expected to type something, and you want them to know what they're expected to type. You can word this however you want. This project's requirements do not specify the wording of any prompts, and the tests will not check your prompts.
    • Declare a new variable to hold the input value and initialize it by calling the appropriate input function (in this project, we will only use nextLine and nextInt).
  2. Test your program. You should see the first input prompt, and then the program should pause until you enter your input before moving to the next prompt. After all of the prompts finish, you'll be left in Jshell. Try entering the name of each variable you created. You should see the values you typed earlier when the program was asking you for input.
  3. In the calculations section, calculate the character's HP (hit points), AC (armor class), and attack bonus. Each of these properties should be a new variable. Test your program after you finish and check the values of the hp, ac, and attack bonus variables. Do the calculations yourself to confirm the values shown in jshell are correct. Use the following formulas to calculate these properties:
    • The character's HP is equal to their level multiplied by their constitution score.
    • The character's AC is equal to ten plus their dexterity score.
    • The character's attack bonus is equal to their level plus their strength score.
  4. Print all of the information shown in the I/O Format section. You can insert your variable into the output using string concatenation: write the label as a string literal, then add the appropriate variable. For example, if your player's name is in a variable called playerName, then you could print "Player: " + playerName for the first line of the character sheet.
  5. Before you run my test program, try running your program yourself and make sure the output looks correct. It will be nice to see the whole program working, and you can spot the more obvious mistakes on your own.

Task 2: Run Tests

See the testing chapter if you don't know how to use the test program.

Run the tests for your project and look at the feedback. If your code passed every test, then this Guided Project is finished. The test program can pack all of your projects for upload to Blackboard.

Deliverables

  • Your CharacterSheet1.jsh file

In order for a guided project to count as complete, it must:

  • be submitted on Blackboard before the final deadline
  • packaged and named correctly (use the test script's pack command)
  • pass all of the tests I gave you

It should be easy to tell whether your project is complete as long as you run the tests. If it's not passing and you can't figure out why, then ask for help rather than submitting an incomplete assignment.

Additional Details

I/O Format

A Guided Project program must meet the following I/O requirements:

  • The program accepts the required input in the order shown in the assignment and does not expect any additional input.
  • The program produces each line of required output in the order specified in the assignment.

The program may include additional output before, between, and after lines of required output. This additional output must be on a separate line from any required output.

Your program must accept the following user input:

  • The player's name (String)
  • The character's name (String)
  • The character's level (int)
  • The character's strength score (int)
  • The character's dexterity score (int)
  • The character's constitution score (int)

Your program must print the following output:

  • Player: <NAME>
  • Character: <NAME>
  • Level: <LEVEL>
  • HP: <HP>
  • AC: <AC>
  • Attack: +<ATTACK BONUS> (note the + before the attack bonus)
  • Str <STRENGTH SCORE>
  • Dex <DEXTERITY SCORE>
  • Con <CONSTITUTION SCORE>
