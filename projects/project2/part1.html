Guided Project 2.1

This program will simulate dice rolls and allow the user to reroll some of those dice.

Learning Objectives

  • Use an arrays to store and process collections of values
  • Use a loop to iterate over an array
  • Use functions to break a program into smaller parts
  • Use functions to reduce code repetition

Prerequisites and Setup

  • Create a file named DiceGame.java in your project2/part1 folder.
  • Download the Dice.java file from OneDrive or copy the code shown below to a file named Dice.java. Add Dice.java to your project2/part1 folder.
/**
 * @author Zach Kohlberg
 *
 * Simulates dice rolls with Math.random(). Includes convenience methods for
 * polyhedral dice (d4, d6, d8, d10, d12, d20).
 *
 * This class is included as starter code so that my tests can replace it with a
 * fake random number generator that supplies a predetermined series of numbers
 * instead of simulating dice rolls. This makes it easier to write test cases,
 * but it means you MUST use this class to generate your random numbers.
 */

public class Dice {
    public static int roll(int sides) {
        return (int)(Math.random() * sides) + 1;
    }

    public static int d4() {
        return roll(4);
    }

    public static int d6() {
        return roll(6);
    }

    public static int d8() {
        return roll(8);
    }

    public static int d10() {
        return roll(10);
    }

    public static int d12() {
        return roll(12);
    }

    public static int d20() {
        return roll(20);
    }
}

Tasks

Task 1: Guided Coding

Follow the steps below and write the program as directed. Test your code frequently, at the bare minimum performing every test given in the steps.

If your code is not working as expected, do not move on to the next step until you have fixed it. Moving on without finishing a step correctly will make the problem significantly harder to fix and likely add new problems as you continue working. If you disregard this advice, then you may get to the end of the steps and find it would actually be less work to start over from step 1 than to debug the code you just finished writing.

Notes:

  • You are required to use the methods in Dice.java to generate all random numbers your program uses. This is necessary for your code to be compatible with some of my tests. You can call any of the methods in Dice.java from your DiceGame.java program by writing Dice.methodName(args).
  • Dice contains a roll method that allows you to simulate a die with any number of sides. You can call Dice.roll(6) in your code to simulate a six-sided die. Dice also contains methods for all of the standard polyhedral dice (d4, d6, d8, d10, d12, and d20). This means you could also use Dice.d6() to simulate a six-sided die. It doesn't matter whether you use the roll method or one of the polyhedral dice methods.
  • This Java programs, like most of the other Java programs you will write, is broken up into multiple methods and multiple classes. There are two ways we will take advantage of this to make our dice game easier to code:
    • When writing a method that performs a complex task, we can simplify our code by breaking the task into smaller sub-tasks and making each sub-task its own method. We can even use these methods before we've finished writing them to test the code for our complex task.
    • When writing a method that performs a smaller task, we can code that method without thinking about the rest of the program as long as we've clearly defined that methods inputs, outputs, and expected behavior. It's much easier to think about a small task on its own than it is to think about the entire program at once.
  • Finally, this project set is meant to get you used to writing methods. My directions will often describe methods and ask you to write them based on that description. I will give you the code for a few of these methods, but you should always try to write them yourself before looking at or using the provided code. This will help you learn what these method descriptions mean so that you're ready to write them yourself when I stop showing you what the code looks like.

  1. We're going to be working with arrays during this project. As you may have already noticed, Java does not print arrays very nicely by default. While it's not required for this program, we can add a temporary printArray method to make testing easier. Try coding printArray based on the following description:

    • a public static method that doesn't return anything.
    • has one parameter: an array of integers
    • contains a for loop that iterates over its array parameter and prints each element of that array
    • (optional but highly recommended) uses print instead of println to keep everything on the same line
    • (optional but highly recommended) ends with a println to move to the next line
    • (optional) includes some kind of header/label before the array so we know what we're looking at in the output
    • (optional) further decorates the output with commas to make it look nice

    If you need help, here's how I would write printArray:

// public, static, returns nothing (void), array of ints as the only parameter (int[] a)
public static void printArray(int[] a) {
    // Print a label for the array and use print (not println) to stay on the same line
    System.out.print("Array: {");

    // Iterate over the array
    for (int i = 0; i < a.length; ++i) {

        // Not important, but I want to include commas and not have an extra trailing or
        // leading comma. It looks a little nicer this way!
        if (i > 0) {
            System.out.print(",");
        }

        // Print the array element with a space before it
        System.out.print(" " + a[i]);
    }

    // Close the curly brackets and move to the next line
    System.out.println(" }");
}

  3. Now it's time to implement the rollAllDice method (remember to delete the placeholder print statement!). This method should iterate over its parameter (the dice array) and set each element equal to a random number from 1 to 6. You already have two methods you can call from the Dice class to generate these numbers, so choose whichever one you prefer. You also have an example of a loop that iterates over an array in the printArray method.

    Once you write this method, compile and run your program to see whether it works. If it's working, then you should only see numbers from 1 to 6 instead of zeroes whenever you print the array.

  4. The rollSomeDice method is similar to the previous method, but we need to reroll the dice listed in our string parameter and skip the rest of the dice. We will assume that the string parameter is a string containing some set of the digits 0 to 4. These represent all of the valid indexes in our dice array. We will only reroll a die if its index appears in that string.

    You can start by copying the code inside your rollAllDice method. Add an if statement to check whether the current index (not the die value) appears in the string parameter. You can ask a string whether it contains another string using the contains method. You'll need to convert your index to a String before calling contains. The method call will look something like this (depending on your variable/parameter names): str.contains("" + index).

    Once you finish this method, go ahead and test your program again! If you enter some numbers from 0 to 4 when asked which dice to reroll, then you should see some of the dice values change when they're printed for the second time. For example, responding with "012" should cause the program to reroll the first three dice in the array. Remember that these rerolls are random, so there's a chance that some of the rerolled dice will end up with the same value as before. However, most of the dice you tell your program to reroll should change.

  5. The final step is to write a diceString method that will convert your dice array to a nice-looking string. We'll show the user the labels for the dice so they know how to specify which dice to reroll, and we'll make the output look more like a row of dice instead of five plain numbers.

    Start by declaring the diceString method: it's a public static method that returns a String and takes an array of integers as its only parameter. Keep the return type in mind: we're not printing anything inside this method body. Instead, we're building a String and returning that string. The code that calls this method will be responsible for printing the resulting string.

    You can check the Additional Details section to see the exact format this string must take. You can use hard-coded strings for the labels and the top/bottom edges of the dice, but you should use a loop to generate the middle of the dice. You can build the middle section by concatenating each element from the dice array with the pipe character and some spaces.

    One last thing to remember is to add line breaks ("\n") between the label row, the top/bottom edges, and the middle section. This won't look very good if everything is on the same line.

    After you finish this method, you should be able to replace your calls to printArray with System.out.println(diceString(dice)). As always, run your program and make sure everything works as expected.

Additional Details

I/O Format

A Guided Project program must meet the following I/O requirements:

  • The program accepts the required input in the order shown in the assignment and does not expect any additional input.
  • The program produces each line of required output in the order specified in the assignment.

The program may include additional output before, between, and after lines of required output. This additional output must be on a separate line from any required output.

Your program must accept the following user input:

  • A "yes" or "no" in response to the first prompt
  • A string of digits from 0-4 in response to the second prompt

Your program must print the following output:

  • The initial set of dice
  • If the user answers yes to the first prompt, then you must also print the dice after the reroll

The dice output should be formatted identically to the diceString output described later on.

Additionally, your program should include reasonable prompts for information and any additional text that you think would make it easier for the user to interact with the program. It's up to you how much of this to include, what exactly to say, and how to format it. The tests will not check these prompts.

Methods

Your DiceGame class must include all of the methods (functions) described below. Each of these methods must have the public and static modifiers.

game

  • Return type: void
  • Parameters:
    • A Scanner for reading user input

Must run through the entire process described in Task 1: Guided Coding:

  • Roll and print a set of five dice
  • Ask the user whether to reroll the dice
  • If the user wishes to reroll, ask the user which dice to reroll, reroll them, and then print the dice again

rollAllDice

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice

Each element of the array parameter must be set to a random number from 1 to 6 generated by either the roll method or the d6 method of the Dice class.

rollSomeDice

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice
    • A String containing digits from 0 to 4, representing the labels of all dice that will be rerolled

Each element of the array parameter whose index appears in the string parameter must be set to a random number from 1 to 6 generated by either the roll method or the d6 method of the Dice class. Each element of the array parameter whose index does not appear in the string parameter should be left as is.

diceString

  • Return type: String
  • Parameters:
    • An int array representing a set of five dice
