CSC110 Projects

Guided Project 2.2

This part adds multiple rerolls and the ability to score a category from the upper section. It also adjusts the program's I/O to slightly improve usability.

Learning Objectives

  • Use an arrays to store and process collections of values
  • Use a loop to iterate over an array
  • Use functions to break a program into smaller parts
  • Use functions to reduce code repetition

Prerequisites and Setup

  • You should have already completed GP 2.1, including debugging the program until it passes all of the tests.
  • Create a copy of project2/part1/DiceGame.java in your project2/part2 folder. Make sure you are working on the file in the part2 folder during this project. Don't leave the file from 2.1 open in another tab in your editor.
  • Copy the Dice.java file to your project2/part2 folder as well.

Tasks

Task 1: Guided Coding

Follow the steps below and write the program as directed. Test your code frequently, at the bare minimum performing every test given in the steps.

If your code is not working as expected, do not move on to the next step until you have fixed it. Moving on without finishing a step correctly will make the problem significantly harder to fix and likely add new problems as you continue working. If you disregard this advice, then you may get to the end of the steps and find it would actually be less work to start over from step 1 than to debug the code you just finished writing.

  1. We'll start with a small change to our previous code. Let's replace the old 0-4 labels with letters:

    • Modify the diceString method to use the labels A, B, C, D, and E instead of 0-4.
    • Modify the rollSomeDice method to look for the letter corresponding to each index rather than the index itself. You can convert an index to a letter by adding it to 'a' and converting the result to a char: (char)('a' + index). You can then convert this character to a String and use it with the contains method like you did in part 1.

    As always, test your program before you continue. Make sure the correct labels are displayed and your program rerolls the proper dice when you enter letters instead of numbers.

  2. We're going to split our game method into two main parts: the reroll phase and the score phase. Most of our current code will move to a method for the reroll phase, and your program should still work the same as before the split. Check that everything is working when you test it.

    To split up the game method:

    • Make a rerollPhase and a scorePhase method. Each of these methods should return nothing and take two parameters: a Scanner and an array of integers (our dice).
    • Move most of the code that's currently in game into rerollPhase. Leave the lines that create the dice array and roll all of the dice in your game method.
    • Call rerollPhase and scorePhase in your game method. The score phase won't do anything for now, but you could add a placeholder print statement just to show that it's getting called. Your game method should look something like this:
public static void game(Scanner input) {
    int[] dice = new int[5];

    rollAllDice(dice);

    rerollPhase(input, dice);

    scorePhase(input, dice);
}

  1. Next, we will use a loop to allow the user to reroll their dice more than once. You could also accomplish this by duplicating a lot of code, but the loop will be shorter, cleaner, and easier to read.

    We want to repeat the reroll process for as long as the user has rerolls left. Make a variable to hold the number of rerolls (it should start at 2), then create a while loop that repeats for as long as the user has more than zero rerolls. Move the rest of your reroll phase code into this loop.

    If the user answers "yes" when asked if they want to reroll their dice, then you should decrease the number of rerolls by one. If the user does not answer "yes", then you can set the number of rerolls to zero so we don't keep asking them to reroll their dice.

    Finally, we can move our code that prints the user's rerolled dice out of the loop. We're already printing the dice at the beginning of the loop, so the user will see what changed between rerolls. We only need to print them once after the loop so they can see the final set of dice before scoring.

    Test your code and make sure you can reroll twice (but no more) if you always answer "yes" when asked whether to reroll dice. Make sure that answering "no" causes the program to print the dice once more before ending.

  2. We're going to make one other change to the reroll phase so that our program is easier to use. Currently, we have to enter two separate messages: one to say that we want to reroll and another to say which dice we want to reroll. As the user, it would be nice to include the dice labels when we tell the program to reroll our dice.

    Modify your loop so that it asks the user for a command instead of asking whether they want to reroll. You should accept two different commands: "roll " followed by some dice labels, and "skip". The roll command should reroll the dice and decrease the reroll count by one, the skip command should set the reroll count to zero, and any other command should result in an error message (see the I/O Format section).

    The roll command might be a little tricky to handle. However, it's much easier once you know the right string methods (check the notes and/or Java API for more information about these methods):

    • You can use startsWith instead of equals to check whether the command begins with the word "roll".
    • You can get a string containing only the labels (without the word "roll") by taking a substring of the command. You'll need to provide the starting index for the substring.
    • You can get the first index where a character appears in a string with the indexOf method.

  3. We need to add a few other methods for use in the score phase: scoreString, calculateDiceScore, and countDice. For now, we're just going to write stubs (placeholders) for these methods. This will let us call them in the score phase code before we finish writing them, much like when we wrote the game method in part 1.

    To create a stub, we just need the method signature line and, if it returns a value, a temporary return statement in the method body. This is enough for us to compile code that calls the method, even if the method doesn't yet do anything useful. Write a stub for the methods listed above. Check the Functions section for details like the parameters and return type of each method. When you add the temporary return statement to these methods, just return "placeholder" or -1 (depending on whether the return type is String or int). We want something that we'll easily recognize in our output so we know that our score phase code called these methods and used their return values.

    As usual, compile your code and make sure it still works before you continue!

  4. With placeholders for our other methods, we can now write the scorePhase method. We'll start with a simplified version that assumes valid input:

    • Call scoreString and print the string it returns
    • Prompt the user to choose which die value they want to score
    • Subtract one from the user's input
    • Call calculateDiceScore with the user's chosen die value and print the result

    Test your code and make sure everything still works. You should now see the output from scorePhase, including the placeholder values returned by some of the method stubs you wrote earlier.

  5. Add a loop to the prompt in scorePhase. If the user enters an invalid number (valid numbers are integers from 1 to 6), then you should print an error message and continue reading input until the user enters a valid number.

    Test your code and make sure it handles invalid numbers correctly.

  6. Write the calculateDiceScore method. You can use the countDice method to count how many times the die value occurs in the dice array, then return that number multiplied by the die value.

    Remember that your second parameter is the category index, not the die value. The die value will be one greater than the category index. This may seem like an odd choice if you follow the user input from scorePhase (where we subtracted one) to this method (where we added one). It will make more sense in project 2.3 when we add the other score categories.

    As always, test your code! We know that our countDice stub currently returns -1, so we should now expect the return value to be whatever die value we enter multiplied by -1.

  7. Write the countDice method. This method should iterate over the dice array and count how many times the given die value appears in the array. When you test your code, you should now see the correct score for the chosen upper category. For example, if the user chose to score 3 with the dice 1, 6, 3, 4, and 3, then they should receive 6 points.

  8. The final method we need to write is scoreString. To start, I recommend creating the following array at the beginning of scoreString: String[] categoryNames = new String[] {"Aces", "Twos", "Threes", "Fours", "Fives", "Sixes"};. This will give you an array with all of the category names and make it much easier to use a loop in this method.

    Now we need to build a string. Remember, we're not supposed to print anything! We need to create and return a string, then let the code that called this method decide what to do with it. Start with the header shown in the I/O Format section: "Score Options:\n".

    Finally, we can create a loop that iterates over each category index. This loop should add a line for that category that looks like this: 1. Aces: 3. We start with four spaces of indentation, then the number the user would enter to choose this category, then the category name (from the array we created earlier), and finally the score we would receive for that category (use the calculateDiceScore method).

    With this method complete, your program should be finished! Test it to make sure everything works. The tests won't give useful feedback if you code can't compile.

Additional Details

I/O Format

A Guided Project program must meet the following I/O requirements:

  • The program accepts the required input in the order shown in the assignment and does not expect any additional input.
  • The program produces each line of required output in the order specified in the assignment.

The program may include additional output before, between, and after lines of required output. This additional output must be on a separate line from any required output.

Your program must accept the following user input:

  • During each iteration of the reroll phase:
    • The command "roll X", where "X" is a series of letters (uppercase or lowercase) from A to E
    • The command "skip"
    • (optional) In addition to "roll X" and "skip", you can accept abbreviations such as "r X" and "s"
  • During the score phase:
    • An integer
    • If the integer is not a valid choice, then read additional integers until a valid choice is entered

Your program must print the following output:

  • During each iteration of the reroll phase:
    • The user's dice, formatted by the diceString method
    • The number of rolls remaining formatted as "X rolls remaining."
    • If the user enters an invalid command, the message "ERROR: INVALID COMMAND"
  • At the end of the reroll phase:
    • The user's dice, formatted by the diceString method
  • During the score phase:
    • A list of options produced by the scoreString method
    • The message "ERROR: INVALID NUMBER" for each invalid input the user provides
    • The number of points received for the chosen category formatted as "You receive X points."

Additionally, your program should include reasonable prompts for information and any additional text that you think would make it easier for the user to interact with the program. It's up to you how much of this to include, what exactly to say, and how to format it. The tests will not check these prompts.

Functions

Your DiceGame class must include all of the methods (functions) described below. Each of these methods must have the public and static modifiers.

game

  • Return type: void
  • Parameters:
    • A Scanner for reading user input

Must perform the following process:

  • Roll a set of five dice
  • Call rerollPhase, passing the scanner and dice as arguments
  • Call scorePhase, passing the scanner and dice as arguments

rollAllDice

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice

Each element of the array parameter must be set to a random number from 1 to 6 generated by either the roll method or the d6 method of the Dice class.

rollSomeDice

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice
    • A String containing letters from a to e, representing the labels of all dice that will be rerolled

Each element of the array parameter whose label appears in the string parameter must be set to a random number from 1 to 6 generated by either the roll method or the d6 method of the Dice class. Each element of the array parameter whose label does not appear in the string parameter should be left as is.

Labels are not case-sensitive. An element's label is determined by its index:

  • index 0 -> a
  • index 1 -> b
  • index 2 -> c
  • index 3 -> d
  • index 4 -> e

diceString

  • Return type: String
  • Parameters:
    • An int array representing a set of five dice

The returned string must be formatted as shown below. Each X must be replaced with an element from the array parameter, and these elements must appear in order.

  A     B     C     D     E
+---+ +---+ +---+ +---+ +---+
| X | | X | | X | | X | | X |
+---+ +---+ +---+ +---+ +---+

rerollPhase

  • Return type: void
  • Parameters:
    • A Scanner for reading user input
    • An int array representing a set of five dice

The user begins with 2 available rolls. For as long as the user has more than 0 rolls, this method must repeat the following process:

  • Display the user's dice and roll count
  • Prompt the user for and read a command
  • Execute the user's command:
    • roll: the string of letters after "roll " must be passed to the rollSomeDice method along with the dice array, and the roll count should be decreased by one
    • skip: the roll count should be set to zero
    • Any other commands must result in an error message

Display the user's dice again after the above loop is finished.

scorePhase

  • Return type: void
  • Parameters:
    • A Scanner for reading user input
    • An int array representing a set of five dice

Must perform the following process:

  • Display the score options and how many points each option is worth
  • Read integers from the user until they enter a number from one to six, displaying an error message for each invalid input
  • Calculate and print the score for the category chosen by the user

countDice

  • Return type: int
  • Parameters:
    • An int array representing a set of five dice
    • An int representing the die value to count

Returns the number of times the die value appears in the array.

calculateDiceScore

  • Return type: int
  • Parameters:
    • An int array representing a set of five dice
    • An int representing the index of a score category (assume a number from 0 to 5)

Returns the score for the upper section category with the given index:

  • 0 -> Aces
  • 1 -> Twos
  • 2 -> Threes
  • 3 -> Fours
  • 4 -> Fives
  • 5 -> Sixes

scoreString

  • Return type: String
  • Parameters:
    • An int array representing a set of five dice

The returned string must be formatted as shown below. Each X must be replaced with the score the user will receive from their dice if they choose that category.

Score Options:
    1. Aces: X
    2. Twos: X
    3. Threes: X
    4. Fours: X
    5. Fives: X
    6. Sixes: X
