CSC110 Projects

Expanded Project 2.3

After completing this part, you should have a fully functioning dice game that you can play on your terminal.

Learning Objectives

  • Use an arrays to store and process collections of values
  • Use a loop to iterate over an array
  • Use functions to break a program into smaller parts
  • Use functions to reduce code repetition

Prerequisites and Setup

  • You should have already completed GP 2.1 and GP 2.2, including debugging the programs until they pass all of the tests.
  • Create a copy of project2/part2/DiceGame.java in your project2/part3 folder. Make sure you are working on the file in the part3 folder during this project. Don't leave the file from 2.2 open in another tab in your editor.
  • Copy the Dice.java file to your project2/part3 folder as well.

Tasks

Task 1: Determine Your Rules

You have two options for this project. You can either implement most of the standard Yahtzee rules, or you can make up your own rules and modify the game.

Option 1: Standard Yahtzee Rules

If you choose this option, then you must complete the program by implementing the remaining parts of Yahtzee. Your program must follow all of the standard rules except for the "Yahtzee Bonus" rules in the official rules. In your program, a Yahtzee should be treated like any other scoring category with no special cases.

This option also requires your program to pass a set of tests, just like a guided project. The additional details section has a list of I/O requirements and function requirements that will be checked by the tests. The structure provided by these requirements may also be helpful if you're having trouble writing this program without the usual guided coding steps.

Option 2: Custom Dice Game Rules

If you are going to change the rules, then you must send me a proposal with your new ruleset. Explain your rules in enough detail that I or another person could play your game with a set of physical dice using only the information in your proposal.

This option exempts you from the tests for 2.3, but it requires an approved proposal at least one week before you submit the project. If you don't meet this requirement, then your project will be reviewed as though you chose the first option.

Task 2: Implement Your Game

If you are implementing standard Yahtzee rules, then refer to the additional details section for specific requirements and the official rules for the actual game rules.

If you are implementing your own custom game, then you should refer to your proposal for specific requirements.

Additional Details (Standard Yahtzee Rules Only)

This section covers the requirements for anyone who chose to implement the standard Yahtzee rules. If you went with the custom rules option, then you can ignore this part of the assignment.

One quick note about category numbers: you may notice that the user's input and the output we display to the user always labels the score categories from one to thirteen, but in our code we use the numbers zero to twelve. The numbers one to thirteen will feel more natural to the average user, but these numbers are all off by one if we need to index an array. Remember to subtract one when going from user input to an array index and add one when going from an array index to output displayed to the user.

I/O Format

The I/O requirements work just like a guided project, and they will be evaluated by the same test program.

Your program must accept the following user input:

  • During each iteration of the reroll phase:
    • The command "roll X", where "X" is a series of letters (uppercase or lowercase) from A to E
    • The command "skip"
    • (optional) In addition to "roll X" and "skip", you can accept abbreviations such as "r X" and "s"
  • During the score phase:
    • An integer
    • If the integer is not a valid choice, then read additional integers until a valid choice is entered

Your program must print the following output:

  • At the beginning of each round:
    • The round number, formatted as "Round X"
  • During each iteration of the reroll phase:
    • The user's dice, formatted by the diceString method
    • The number of rolls remaining formatted as "X rolls remaining."
    • If the user enters an invalid command, the message "ERROR: INVALID COMMAND"
  • At the end of the reroll phase:
    • The user's dice, formatted by the diceString method
  • During the score phase:
    • The output of the scoreString function
    • The message "ERROR: INVALID CATEGORY" for each input the user provides that isn't a number from 1 to 13
    • The message "ERROR: CATEGORY UNAVAILABLE" for each input the user provides that corresponds to a category that was already taken
    • The number of points received for the chosen category and that category's name formatted as "You receive X points for Y."
  • After the final round:
    • The line "Final Score Card:"
    • The output of the scoreString function

Additionally, your program should include reasonable prompts for information and any additional text that you think would make it easier for the user to interact with the program. It's up to you how much of this to include, what exactly to say, and how to format it. The tests will not check these prompts.

Functions

Your DiceGame class must include all of the methods (functions) described below. Each of these methods must have the public and static modifiers.

game

  • Return type: void
  • Parameters:
    • A Scanner for reading user input

Must perform the following process for each round of the game (total of 13):

  • Print the round number
  • Roll five dice
  • Call rerollPhase, passing the scanner and dice as arguments
  • Call scorePhase, passing the scanner, dice and other data as arguments

Must print the final score card after the last round ends.

You should create three additional arrays at the start of this method: the category names (String[]), scores (int[]), and categories used (boolean[])

rollAllDice

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice

Each element of the array parameter must be set to a random number from 1 to 6 generated by either the roll method or the d6 method of the Dice class.

rollSomeDice

  • Return type: void
  • Parameters:
    • An int array representing a set of five dice
    • A String containing letters from a to e, representing the labels of all dice that will be rerolled

Each element of the array parameter whose label appears in the string parameter must be set to a random number from 1 to 6 generated by either the roll method or the d6 method of the Dice class. Each element of the array parameter whose label does not appear in the string parameter should be left as is.

Labels are not case-sensitive. An element's label is determined by its index:

  • index 0 -> a
  • index 1 -> b
  • index 2 -> c
  • index 3 -> d
  • index 4 -> e

diceString

  • Return type: String
  • Parameters:
    • An int array representing a set of five dice

The returned string must be formatted as shown below. Each X must be replaced with an element from the array parameter, and these elements must appear in order.

  A     B     C     D     E
+---+ +---+ +---+ +---+ +---+
| X | | X | | X | | X | | X |
+---+ +---+ +---+ +---+ +---+

rerollPhase

  • Return type: void
  • Parameters:
    • A Scanner for reading user input
    • An int array representing a set of five dice

The user begins with 2 available rolls. For as long as the user has more than 0 rolls, this method must repeat the following process:

  • Display the user's dice and roll count
  • Prompt the user for and read a command
  • Execute the user's command:
    • roll: the string of letters after "roll " must be passed to the rollSomeDice method along with the dice array, and the roll count should be decreased by one
    • skip: the roll count should be set to zero
    • Any other commands must result in an error message

Display the user's dice again after the above loop is finished.

scorePhase

  • Return type: void
  • Parameters:
    • A Scanner for reading user input
    • An int array representing a set of five dice
    • An int array containing the score for each category
    • A boolean array representing whether each category has been used
    • A String array containing the name of each category

Must perform the following process:

  • Display the score options, their availability, and how many points each option is worth
  • Read integers from the user until they enter a number from one to thirteen that corresponds to an available category, displaying an error message for each invalid input
  • Calculate and print the score for the category chosen by the user
  • Store the score in the scores array
  • Mark the category as unavailable

countDice

  • Return type: int
  • Parameters:
    • An int array representing a set of five dice
    • An int representing the die value to count

Returns the number of times the die value appears in the array.

totalScore

  • Return type: int
  • Parameters:
    • An int array containing the score for each category

Returns the total score for all categories, including the upper category bonus if applicable.

upperTotal

  • Return type: int
  • Parameters:
    • An int array containing the score for each category

Returns the total score for the upper categories, not including the upper category bonus.

upperBonus

  • Return type: int
  • Parameters:
    • An int array containing the score for each category

Returns the upper category bonus: 35 points if the upper category total is at least 63, or 0 points if it is not.

lowerTotal

  • Return type: int
  • Parameters:
    • An int array containing the score for each category

Returns the total score for the lower categories.

calculateDiceScore

  • Return type: int
  • Parameters:
    • An int array representing a set of five dice
    • An int representing the index of a score category (assume a number from 0 to 12)

Returns the score for the category with the given index:

  • 0 -> Aces
  • 1 -> Twos
  • 2 -> Threes
  • 3 -> Fours
  • 4 -> Fives
  • 5 -> Sixes
  • 6 -> Three of a Kind
  • 7 -> Four of a Kind
  • 8 -> Full House
  • 9 -> Small Straight
  • 10 -> Large Straight
  • 11 -> Yahtzee
  • 12 -> Chance

The score is calculated using the given set of dice.

calculateUpperCategory

  • Return type: int
  • Parameters:
    • An int array representing a set of five dice
    • An int representing the index of a score category (assume a number from 0 to 5)

Returns the score for the upper section category with the given index. The score is calculated using the given set of dice.

Lower Categories (multiple functions)

  • Function names: calculateThreeOfAKind, calculateFourOfAKind, calculateFullHouse, calculateSmallStraight, calculateLargeStraight, calculateYahtzee, and calculateChance
  • Return type: int
  • Parameters:
    • An int array representing a set of five dice

Each of these functions returns the score for its category. The score is calculated using the given set of dice.

scoreString

  • Return type: String
  • Parameters:
    • An int array containing the score for each category
    • A boolean array representing whether each category has been used
    • A String array containing the name of each category
    • An int array representing a set of five dice

The returned string must be formatted as shown below.

Each X after a category name must be replaced with either:

  • If the category is available: "available (Y)", where Y is the score the user will receive from their dice if they choose that category.
  • If the category is used: the score previously assigned to that category.

The X after "Upper Section", "Upper Section Bonus", "Lower Section", and "Total" must be replaced the the appropriate total/bonus.

Each category's line should be indented with four spaces.

Upper Section: X
    1. Aces: X
    2. Twos: X
    3. Threes: X
    4. Fours: X
    5. Fives: X
    6. Sixes: X
Upper Section Bonus: X
Lower Section: X
    7. Three of a Kind: X
    8. Four of a Kind: X
    9. Full House: X
    10. Small Straight: X
    11. Large Straight: X
    12. Yahtzee: X
    13. Chance: X
Total: X
