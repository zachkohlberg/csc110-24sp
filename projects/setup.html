Project Directory Setup

Before you start working on any projects, you should create folders to store your project files. I would like you to use a specific structure for a few reasons:

  • I will give you test code for the Guided Projects. This test code expects your projects to be organized a certain way. You may not be able to run the tests if you don't organize your projects as shown below.
  • When I download projects from Blackboard, I process them automatically with some scripts. This saves me hours of organizing and renaming everyone's files throughout the semester. My scripts also handle testing the Guided Projects, and they'll use the same test code I give you. If your submission isn't organized correctly, then my scripts will not be able to extract the project files and run the relevant tests, and your submission will be marked incomplete.
  • Keeping your work organized will make it easier for you to keep track of everything.

Project Directory Structure

Your project directory should look like this:

projects/
├── final/
├── project1/
│   ├── part1/
│   ├── part2/
│   └── part3/
├── project2/
│   ├── part1/
│   ├── part2/
│   └── part3/
├── project3/
│   ├── part1/
│   └── part2/
└── tests/

Everything you need to submit should go in the folder for that project. For example, your code for Guided Project 1.1 would go in the folder projects/project1/part1. Each folder name must only contain lowercase letters, and the spelling must match exactly what is shown above. Ignore the / at the end of each name; this is to indicate that the name refers to a directory rather than a regular file.

You can create these directories by hand from your file browser, but it's easier to use the terminal. The following commands create all dozen or so folders with the correct names, and they will work in most Mac/Linux terminals. If you're on Windows, then read the terminal section of the notes and follow the directions for getting a terminal emulator that can run bash commands. The default Windows terminals (cmd and powershell) will not be able to run most bash commands.

mkdir projects
cd projects
mkdir -p final project{1,2,3}/part{1,2} project{1,2}/part3

The name of the root folder (projects) doesn't technically matter, so you can use a different name. Just change the name projects in the first two commands to your new folder name, but make sure it doesn't contain any spaces.

You don't need to create the tests folder right now. It will be created when you download the test code and extract the contents.
