CSC110 Syllabus

Course Material

Primary Resources

  • Notes: I'm in the process of writing and revising notes for all of the topics we will cover in this class. They're available in HTML (recommended version, linked on Blackboard) and as a PDF (also on Blackboard). You can read these at your own pace, but at a minimum you should complete the assigned readings to prepare for each week's classes. I will update the notes frequently as I complete revisions and add new material.
  • Class Activities/Examples: We will often write short programs during class, either as part of a demo or when reviewing a group activity. I will post all of these programs in a shared OneDrive folder linked on Blackboard.

Supplemental Resources

  • Videos: I plan to record some videos covering how to set up and use certain software, how to set up and submit the guided projects, and possibly other topics. These are a lower priority than most of my other prep work, so the number of videos depends on how much extra time I have during the semester.
  • Textbooks: You do not need to purchase any textbooks for this class, but I recommend using a textbook or other reliable source to read about the topics we cover in more detail. I can recommend the book Programming and Computation Fundamentals, by Busbee and Braunschweig, which is available for free from <libretexts.org>. My notes may refer to this book from time to time.

Software

  • Text Editors: You need a text editor to edit code (Microsoft Word and Google Docs are not text editors). I've listed a few options below. I recommend picking one editor that you like and learning to use it well. Get used to using your editor's keyboard shortcuts and you'll be able to work very fast.
    • VS Code: If I had to guess, I'd say this is the most popular editor currently available. I recommend using it unless you have a good reason to go with another editor. One caveat: I do not use VS Code and have very little experience with it, and I cannot provide technical support if you have trouble using it. However, due to its popularity you should be able to find plenty of resources online.
    • Neovim: This is the editor I use. It's harder to learn than the other options in this list, but it's powerful and highly configurable. If you want to spend more time initially learning vim motions and various keyboard shortcuts, then it might be worth giving neovim a try (or vim or emacs, which are both similar to neovim). This might also be a good option if you like the idea of gradually customizing your editor to suit your exact preferences. However, I think most people will be happier with VS Code, so I don't recommend Neovim unless it really appeals to you and you're fine with the learning curve.
    • Notepad++: I like Notepad++, but it's Windows only and I haven't used it in years. It still appears to be receiving regular updates, so it should be a solid choice if you're using Windows and don't like VS Code.
    • Sublime Text: I've never used Sublime, but it was somewhat popular when I was a student. Like Notepad++, it still appears to be receiving updates.
  • Java: This is the programming language we're using. You'll set up Java during the first lab, and I'll post directions on Blackboard in case you need to reinstall it in the future.
  • Git Bash: We'll be using the terminal regularly. I don't like the default terminal programs on Windows (cmd and Powershell), and some of the scripts and commands I give you will not work in either of them. Git for Windows comes with a decent terminal emulator, and I highly recommend it over cmd and Powershell if you use Windows. If you're using a Mac or Linux computer, then the default terminal program should be fine.
  • diagrams.net: This is like Google Docs for flowcharts and other diagrams. It can also integrate with Google Drive and allow multiple people to edit the same diagram, but some students have reported issues with lag while doing this.

Other Resources

Asking good questions is one of the best skills you can have as a student. In my experience, very few people (myself included) ask questions as often as they should, and it can be difficult to ask a question that will quickly lead to a helpful answer. The blog posts below provide excellent advice for asking and answering questions. I recommend giving them a quick read, especially if you have trouble asking questions or often feel like the answers you get aren't helpful.

  • https://jvns.ca/blog/good-questions/
  • https://jvns.ca/blog/2021/10/21/how-to-get-useful-answers-to-your-questions/
  • https://jvns.ca/blog/answer-questions-well/

The Java API is a useful reference. It may be intimidating right now, but the more you learn about Java and programming, the more easily you'll be able to navigate the API and use it to look up information.

  • Link: https://docs.oracle.com/en/java/javase/21/docs//api/index.html
  • You can also Google "java api" and whatever part of the API you're trying to use, and this will usually give you the right page as the first result. You may get the documentation from older versions of Java if you do this, but most of that documentation is still accurate for more recent versions.
