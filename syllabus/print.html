Description

CSC110 is an introduction to programming using the Java programming language and the Jshell REPL. The primary focus of this class is developing problem-solving skills and a general knowledge of programming that will be useful to you in most programming languages. While we will spend a lot of time learning about Java and how to write good code, writing code isn't our ultimate goal. The most important skill you can learn in this class is how to think like a programmer. Once you've mastered that, writing code is surprisingly easy.

This syllabus also covers any sections of CSC110L that I am teaching this semester. Policies specific to CSC110L are in the lab section of this syllabus. If you are taking the lab with a different instructor, then refer to their lab syllabus.

Objectives

CSC110 and CSC110L share objectives. Upon successful completion of both courses, students should be able to:

  • Discuss the challenges inherent in communicating with (programming) a (non-reasoning) computer.
  • Break a complex problem down into smaller more manageable components.
  • Provide step-by-step instructions to solve small computational problems.
  • Using at least one programming language, write examples of and solve problems in programs with the following basic programming constructs and tools:
    • Mathematical expressions
    • Conditional expressions
    • Simple iterative statements
    • Simple functions
    • Simple data structures, such as arrays, lists, and strings
  • Discuss the importance of commenting and good code structure/style

    • Notepad++: I like Notepad++, but it's Windows only and I haven't used it in years. It still appears to be receiving regular updates, so it should be a solid choice if you're using Windows and don't like VS Code.
    • Sublime Text: I've never used Sublime, but it was somewhat popular when I was a student. Like Notepad++, it still appears to be receiving updates.
  • Java: This is the programming language we're using. You'll set up Java during the first lab, and I'll post directions on Blackboard in case you need to reinstall it in the future.

Assignments

There are three types of assignments in the grade contracts:

  • Guided Projects
  • Expanded Projects
  • Final Project

We will also have reading assignments and other homework that isn't in your grade contract. These are never graded, but they will usually prepare you for an activity in the next class meeting.

Guided Projects

Most of our projects will be larger programming problems broken into two-three parts. The first two parts are called "Guided Projects". The Guided Projects contain step-by-step instructions, further breaking down their part of the program into a series of small tasks. My goal for each Guided Project is to have you practice solving many small problems while building up to a more complex and interesting program. I've designed most of the program for you, but you still need to figure out how to code each part using the information provided.

Expectations

Guided Projects require you to write and debug code. When working on these projects, you should:

  1. Follow the steps, completing the tasks in the order they are given.
  2. Test your code as often as possible, especially when directed to in the assignment.
  3. Finish the current step and make sure it works correctly before moving on to the next step. Moving on when you haven't fixed an error will make the error harder to fix, make the next step more difficult, and lead to more mistakes on the rest of the project. This is why frequent testing is important.
  4. Run the test program included with the assignment after you've completed all of the steps. If your work does not pass every test, then fix your mistakes and re-test it until it passes.

I want to emphasize that both writing and debugging your code are equally important parts of these projects. All programmers make mistakes, and debugging is a crucial skill to learn. Be patient, take the time to learn why code isn't working before you attempt to fix it, and resist the temptation to take shortcuts.

Review

I review Guided Projects by running the test program included with the assignment. If your submission is packaged correctly and passes all tests, then it is complete. I will try to review all new guided project submissions weekly and update Blackboard to show which projects are complete.

Expanded Projects

Expanded Projects build on the program you wrote in the guided projects that came before it. Instead of step-by-step instructions, you will have several open-ended tasks to complete. These tasks will be more complicated than the Guided Projects, and more of the details will be left up to you. Expanded Projects don't normally include tests like the Guided Projects.

Each Expanded Project will also include a write-up. The write-up allows you to show me how you designed parts of the program, reflect on the code you wrote, and demonstrate your understanding of your program. I consider this to be roughly as important as your code, so please put an appropriate amount of time and thought into the written portion of your project.

Proposals

You can propose a different set of tasks to replace the tasks I gave you in the assignment (except for the write-up). Your alternative tasks must still meet the Learning Objectives given at the beginning of the assignment. I will provide a proposal template to use if you would like to modify an Expanded Project.

If you are modifying an Expanded Project, then you must get your proposal approved at least one week before you submit the project. This is to encourage you to write the proposal early and before starting work on the project. Keep in mind that you may need to revise the initial proposal before I approve it, so it's best to submit proposals at least two to three weeks before the project deadline.

Expectations

Expanded Projects require you to write and test code, but with less structure than the Guided Projects. You will need to spend more time designing the code you write and consider how it affects other parts of the program. You will also need to write about your work, explaining and assessing the changes you made to the program.

I expect you to complete every programming task and address every writeup question before you submit your work. I do not expect your work to be perfect, but I do expect it to be finished and result in a runnable program. If you are confused by some of the directions, then you need to ask me about them as early as possible.

Review

I will read and test your submission before providing feedback. If you completed every task as instructed, fully answered every writeup question, and included every deliverable with your submission, then the assignment is complete. It doesn't matter if I disagree with your opinions in the write-up or think you should have written part of the program differently. All of my feedback in the project review will be aimed at helping you improve as a programmer, and it has no bearing on any grade.

If you skip any tasks, forget to include a deliverable, or do not fully address each writeup question, then your work is incomplete. I will provide minimal feedback to indicate what part(s) of the assignment are missing, and you can resubmit the assignment. If you resubmit after the deadline, then you will incur the usual late penalty specified by your grade contract. An incomplete submission, even if accidental, does not entitle you to extra time. If you are not confident that you have followed the assignment directions correctly, then you can always ask me to check your work during office hours or submit a week or two early to leave time for a revision.

Final Project

The Final Project is an open-ended project and works similarly to an Expanded Project. However, there are a few differences:

  • You will start from scratch and design the entire program.
  • You must submit a proposal (as you would for alternative tasks in an Expanded Project), and you must receive approval at least a week before you can turn in the assignment. The Final Project proposal will include additional information about the Learning Objectives your project should cover.
  • You can work in a group, but group projects will have additional requirements. I will also expect them to be larger in scope than a typical solo project.

Contract Grading

Contract grading is a form of "labor-based grading." You can find plenty of articles about labor-based grading and grade contracts if you want to search for it. Here's one that was easy to find and that I think explains the concept well: https://cep.barnard.edu/labor-based-grading. There's also a link to an example contract in that webpage. My implementation will differ a little from what they describe, but my goals are mostly the same.

Why Grade Contracts?

The author of the example grade contract shown in the Bernard College webpage gives an excellent explanation of why they prefer contract grading over traditional grades. I recommend reading that if you want a more thorough explanation than what I've written here.

I'll try to summarize why I dislike traditional grading systems and think grade contracts address some (but not all) of the problems with grades.

  • Grades are not a good source of feedback. They aren't necessary for feedback, and they don't tell you much. I'd rather write comments explaining what I think you did well, pointing out mistakes and areas for improvement, and suggesting how you can improve.
  • The grade contracts are more transparent than traditional grades. You know exactly which assignments you need to complete to receive a particular grade. If I've designed and written my assignments well, then you'll know exactly what I expect and can determine for yourself when you've met those requirements.
  • The grade contract deemphasizes grades. I don't grade individual assignments. I need to determine if you completed the assignment as instructed, so I am still enforcing a standard, but beyond that I am not penalizing you based on the quality of your work. You are a student, not a professional software developer, and I expect you to make a lot of mistakes. This is a necessary part of learning, and I don't want to punish you for it with lower grades. I'd rather encourage mistakes and help you learn from them.

One other note: graded exams, quizzes, and similar assessments don't work well with grade contracts, or at least not with the style of contract I'm using. This means we won't have any exams or quizzes in this class, with the possible exception of ungraded practice quizzes. This also means there is no midterm/final exam.

Modifying a Grade Contract

Most students will use one of the default contracts. However, you may want or need to alter your contract in some way. In that case, discuss your proposed changes to the contract with me before you submit the modified contract. You can submit a modified contract as your initial grade contract, or you can ask to change to a modified contract at any point during the semester. I am under no obligation to approve your modifications, but I will try to work with you to find a compromise that works for both of us.

If you make changes to the default grade contract, then you must explain these modifications in your contract and clearly mark all changes (for example, by highlighting them). If you do not do this, then your modified contract is invalid and I will hold you to the default contract for your chosen grade.

Some reasons you may want to alter the default contract:

  • You need an accommodation that conflicts with one of the contract requirements or one of the policies in this syllabus.
  • You want to replace an assignment from the contract. You may dislike a particular assignment or have a different idea that you think would be more interesting to you. Whatever the reason, I'm open to replacing assignments or groups of assignments on a case by case basis. Keep in mind that it will be up to you to give me a detailed proposal for the alternative assignment and demonstrate that it meets the same objectives as the assignment(s) it would replace. This option will often entail more work from you, but if it makes the projects more fun then it's probably worth that extra effort.

A few examples where a modified contract would not be appropriate:

  • You want to remove an assignment from the contract without replacing it. If you want less work, then you can choose a contract that requires less work. If you're not even willing to do the guided projects in the C contract, then you should drop this class.
  • You want to propose an A- or other compromise contract that just cuts some of the assignments from a higher contract. The contracts already specify grade penalties for skipping certain assignments.
  • You want to add extra credit to make up for incomplete or late work. It makes no sense to skip assignments and then propose extra work to do instead. Just do the work in the contract or propose an alternative before that work is due.
  • You wish to be exempt from a class policy but do not have a compelling reason for the exemption.

If you want to modify the grade contract but aren't sure whether I would allow your proposed changes, just ask me about it.

Additional Notes

Some additional information about the grade contracts that should answer common questions.

  • There is no 100/A+ contract. The highest letter grade you can earn at Quinnipiac is an A, not an A+ (https://catalog.qu.edu/academics/grading-system/). The reason the contract specifies a 95 is so that the penalties have a consistent impact in each of the three contracts. The actual number doesn't go on your transcript, just the letter grade.
  • I do not round grades, which may matter if you lose a fraction of a point somewhere. For example, a 72.75 does not meet the minimum 73 for a C and would therefore be a C-.
  • There is no extra credit. You choose what grade you want and you know what work is required. Do that work and you get that grade. Don't do that work and you get a lower grade. That's how this system works.
  • There is no partial credit for partially-finished assignments, and I will not change the late penalties because you were almost on-time or almost within the 48-hour late window. You must complete the entire assignment for it to count, and you must submit it on time to avoid a late penalty. If you're having trouble, then please ask me or a tutor for help!
  • I tried to set the penalties and deadlines so that it will always make sense to prioritize the Guided Projects over the harder assignments if you are struggling to finish all of your work. They cover the essential skills we want you to learn in CSC110, so if you only do some of the assignments then you should do the Guided Projects.

If you have questions about the grade contracts that aren't answered in the syllabus, or if you find something in them unclear or confusing, please don't hesitate to ask me about it! Most students aren't used to this type of grading system, so I want to address all of your questions and concerns before you submit a contract.

Late Work and Extensions

The latest I will accept any assignment submissions is the last day of classes shown in the Academic Calendar. If an assignment is not complete and submitted before this deadline, then the assignment will count as incomplete on your grade contract.

Some assignments may have deadlines before the last day of classes. The contracts specify a late penalty for missing this deadline, and you cannot submit them at all after the last day of classes.

Some assignments do not specify a deadline, but they may have other constraints. For example, you may need to give a presentation during a class meeting as part of an assignment. If you miss your last opportunity to finish such an assignment, then it will count as incomplete on your grade contract.

Extensions

In rare cases, I will allow extra time to complete an assignment. You can email me to ask for a deadline extension if you believe one is appropriate in your situation. Your email should include an explanation of why you need the extension, a new proposed deadline, and your current progress on the assignment. Be sure to send me your code and any other material you've created so far.

I have some expectations if you are requesting an extension:

  • You've demonstrated that you read this policy by including the information I asked for.
  • If you're having trouble with the assignment, then you're taking steps to get help. Most likely this would include seeing a tutor and/or going to office hours.
  • If it's close to (within a week of) the deadline, then you have already finished the majority of the assignment.
  • If it's after the deadline, then you have a compelling reason that you were not able to contact me earlier and can provide some form of documentation to support this.

I will not extend any deadlines past the last day of classes unless I am giving you a final grade of Incomplete as defined in the university's grading system.

Advice

If you struggle with deadlines or time management, then I can offer some advice:

  • Start your work early. You have six to nine assignments to complete depending on your grade contract, and you probably won't be ready to start them until week 3. That's one to two weeks per assignment. It's not a big workload if you spread it out, but it will be overwhelming if you let it pile up.
  • Deadlines in this class are not targets. Time estimation is difficult, and it's easy to underestimate how long an assignment will take. If you plan to submit something on the due date, then you may be unable to finish on time. Aim to submit work at least a week before it is due, and then use the extra week to deal with unexpected problems.
  • There are people you can talk to if you need help. Talk to them when you start having trouble; don't wait until you feel hopelessly stuck or overwhelmed. If you do start feeling overwhelmed or hopelessly stuck, then it's even more important that you talk to someone right away!
    • I'm available in person during office hours and by email or appointment (for remote meetings) outside of office hours.
    • The Learning Commons employs peer tutors who can help you with this class.
    • Your Academic Coach can be reached by email and is available by appointment. They may not be able to help you with your code, but they're a great person to talk to if you're struggling with time management or adjusting to a college environment.
  • Keep backups of your work in case you have a problem with your computer. Google Drive and Dropbox both have free plans, and I believe you can access OneDrive through the school.

Academic Integrity (AI) Policy

This section assumes you understand the school's AI policy. I want to cover some important details about how it applies in our class.

Permission to Share and Discuss Work

I consider it very important that you feel comfortable discussing assignments with other students. Your peers can be an excellent source of help and feedback, and your own insights and feedback can be incredibly valuable to them. To ensure that the school's AI policy does not penalize you for this, I give you explicit permission to do the following:

  • You may share any work you create for this class with anybody, including your classmates.
  • You may publicly post work you create for this class.
  • You may use work you originally created for this class in an assignment for another class as long as you get the other instructor's explicit written permission first.
  • You may discuss assignments from this class with anybody. You may ask anybody for help with them and view their work if they're willing to share it with you.

In summary, you own all the work you do for this class, and that means you can decide what you do with it. However, I need to clarify that this permission does not extend to the following:

  • You may not share work or post it publicly for the purpose of helping someone else violate the school's AI policy. This includes if somebody asks you for your work and informs you that they plan to copy it before you share the work with them. This type of action is considered "facilitation" under the school's AI policy.
  • You may not use work that you originally created for another class in any of your assignments for this class without the explicit permission of both me and the other class's instructor. The permission granted previously only applies to reusing work from this class in another class, and you would still need permission from that class's instructor. If you are ever unsure, just ask me and the other instructor.
  • You may not copy your work from other people or from a large language model (an "AI"). I'll cover this in more detail in the next section.

I do not consider you responsible for anyone's misuse of material that you shared in good faith. I will always take your word regarding how you meant for or expected your work to be used unless I have strong evidence that contradicts you. If there's ever a question of whether you had permission to share work from this class, please contact me and I will vouch for you.

What Counts as Plagiarism?

First and most important: in any situation where you have doubts or are unsure whether something is allowed, ask me. It's always okay to ask whether you can do something, and this is the best way to be sure that I won't treat it as an AI violation.

This section exists for two reasons: to make it easier for someone who has genuine questions to determine whether an action qualifies as cheating, and to give me something to point to if somebody claims they didn't know their actions qualified as cheating. Going through a series of flimsy excuses is one of the most tedious parts of handling an AI violation, so I'd like to be thorough here and save myself some trouble later.

The vast majority of AI violations that occur in this class involve plagiarism on programming assignments, so that's what I will focus on. To keep my descriptions short, I'll be using "other work" to refer to any work that was created by somebody other than you or by an LLM (such as ChatGPT). The following actions will nearly always qualify as cheating and/or plagiarism (I'll note exceptions later):

  • Copying the other work in whole or in part, whether by copying the file, copying and pasting the file's contents, or viewing the file and retyping the same information by hand. Modifying the other work after copying it, whether to disguise the plagiarism or "make it your own" does not change the fact that it was copied. What matters is whether you copied work, not how you copied it or what you did before or after copying it.
  • Paraphrasing the other work in whole or in part. This is not meaningfully different from copying, and it's often an attempt to disguise the copying and provide some plausible deniability if caught.
  • Using the other work as a reference when creating your own work. This normally ends up with you copying or paraphrasing as described above, but I mention it because it's another common excuse.
  • Doing your work under the direction of another person. It's okay to talk to someone about your work while you're working, but it's not okay to let them dictate the code to you. If you're mostly writing what someone else tells you to write, then you are not doing your own work. Try talking to them about your problem, take some notes, and then go fix the problem after you're done talking.

Exceptions to the above:

  • You can use any material that I intentionally provide you (whether it's in my notes, starter code, the assignment, or something I showed during class) as-is or with modifications in your own work. I gave you that material so you could use it, and you don't need to remind me about this. You can still document your use of starter code or other resources that I provided if you want to, it's just not necessary when submitting it in this class. If you show the work outside of this class, then you'll probably want to be more explicit about which parts you wrote yourself because that is no longer obvious from context.
  • If you want to copy or paraphrase a portion of work from another source, then you can as long as you do the following:
    • Discuss it with me first and get my permission to use the other work.
    • Clearly state which outside sources you used and provide a reliable link or other means to access those sources.
    • Take great care to show what was taken from the other source and what is your own original work. It should be immediately obvious to anyone viewing your work which parts are from other sources.

Two general rules I use if someone appears to have plagiarized but claims they did not:

  • If large sections of your code are identical or nearly identical to existing code, then it isn't your work. If you are writing your own code and following the guidelines I provide in this policy, then you should always end up with significantly different code on all but the simplest programs.
  • If you don't understand your code and cannot explain how it works, then it isn't your work. You should not be writing code that you don't understand, and you are not going to accidentally write a working program that solves a specific problem without some understanding of the code you're writing. If your code doesn't appear to be something you realistically could have written and you cannot explain how it works, then I will not accept it for an assignment.

Retaking this Class and Self-Plagiarism

If you are retaking this class, then I expect you to complete every assignment from scratch without reusing any of your old work. Reusing work from the last time you took this class would be equivalent to reusing work you completed for a different class. As stated earlier in this policy, you do not have my permission to do this.

ChatGPT and LLMs

You're free to discuss assignments with LLMs such as ChatGPT. However, I do not recommend using them too heavily, and I want to warn you about some of their limitations before I get into how they pertain to academic integrity. Despite the colloquial term "artificial intelligence," these tools are not intelligent and do not understand the conversation they are having with you. They are complex algorithms running on powerful hardware, and they are very good at one thing: using their training data to produce text that looks like something a human might write in the context of your conversation. LLMs are prone to hallucinating (making up false information), and they do not know when they are hallucinating. If you don't understand the topic you're discussing well enough to critically evaluate what ChatGPT tells you, then I think you're better off talking to a human who does understand the topic.

I have a simple rule for how you may use material generated by an LLM: treat anything it writes the way you would treat information you found by searching the internet or information that someone else gave you. It is not your work, and all of the restrictions I've already stated regarding "other work" apply. You should also avoid asking an LLM to do something that you would not be allowed to ask a human to do. Don't ask ChatGPT to rewrite or fix your work for you. This would qualify as cheating. Instead, ask it questions so that you can understand how to fix your work, and then fix it yourself.

If you are using ChatGPT for help with an assignment, then please save that conversation so you can share it with me or the AI Office if your work is called into question. This is an easy way to prove you used the tool appropriately, and if you cannot or will not provide that evidence then I will not give you the benefit of the doubt.

Consequences for AI Violations

Sanctions for AI violations are determined either by joint resolution, case review, or a hearing. In a joint resolution, I must agree to any proposed consequences, and in a case review or hearing I will still be consulted about appropriate consequences. This section covers what I believe are appropriate consequences on a first offense for most AI violations.

If you plagiarize, cheat, or otherwise violate the AI policy with work you submit for an assignment, then under no circumstances will I consider it appropriate to give you any credit for the assignment or allow you to attempt the assignment again. This is the penalty stipulated in the grade contract for not completing the entire assignment, not the penalty for cheating. If you copied or cheated on even a small portion of the assignment, then you did not complete the entire assignment; you completed some of the assignment and then copied the remaining work. To give any credit would be a reward for violating the AI policy.

In addition to enforcing the grade contract's penalty for incomplete work, I consider the following penalties to be appropriate:

  • In cases of cheating on only a small part (let's say under ten percent) of one or two assignments, an additional final grade penalty equal to half of each assignment's incomplete penalty.
  • In more significant cases of cheating on a single assignment, an additional final grade penalty equal to the assignment's incomplete penalty.
  • In cases of cheating on multiple assignments, a failing grade in the class.

These additional penalties cannot be removed by changing your grade contract. If you cheat on a project for the B contract, then the additional penalty will still apply if you change to the C contract.

Lab Assignments

I plan on giving ten lab assignments during the semester (this number was later reduced to eight total assignments). Lab assignments will typically have the following structure:

  • A group activity to complete with other students in the lab, which may involve writing code or producing other material that should be shared among all group members. Each group member should turn in a copy of this group work, ensuring that the names of everyone in the group are listed in each file/document.
  • A written portion to be completed individually. You are free to discuss the questions with your group, but you must write your own answers.

Make-Up Classes

I have scheduled four of our class meetings as make-up classes. You may complete/redo a single missed or incomplete lab assignment per week either during a designated make-up class or during my in-person office hours before the lab. Depending on why you did not receive credit for the assignment, I may require that you redo both the activity and the write-up from the beginning.

I cannot guarantee we will have all four make-up classes. If I need to cancel a class meeting, then I will most likely cut one of the make-up days from our schedule. Lab Assignment Completion

Lab assignments are not graded. Each assignment is marked as either complete or incomplete:

  • A complete assignment submission includes all required materials and demonstrates that you followed all directions during the lab activity. It must be submitted by the end of the class meeting in which it was scheduled or during a make-up class.
  • An incomplete assignment may be missing required materials, may not demonstrate that you completed the activity as directed, or may not have been submitted due to an absence or failure to finish the activity in the allotted time.

Keep in mind that the lab's purpose is to practice applying material from the lecture and to reflect on that material by writing about it. There is no wrong way to do this as long as you complete the entire practice activity and answer every question in the write-up. I don't care about the "correct" code or the "correct" answers to the write-up questions, I only care about your code and your answers to the write-up questions.

Lab Grades

Your grade in the lab is determined by the number of lab assignments you complete:

  • A: eight complete labs
  • B: seven complete labs
  • C: six complete labs
  • D: five complete labs
  • F: four or fewer complete labs

There are no in-between grades and no partial credit for lab assignments; each lab is either complete or incomplete. The highest letter grade you can receive is an A because the university's grading system does not allow for an A+ or any other grade above an A. I did not include numeric scores with the letter grades because I only send a letter grade to the registrar. A numeric score would only matter if it affected your letter grade, which is not possible with this class's grading system.

Unlike the lecture, I do not feel a grade contract makes sense for the lab. However, I have still tried to do what I can to de-emphasize and simplify grades and instead emphasize the learning process. I do not grade individual assignments, and I only consider whether the required work was completed on time.

Academic Integrity

Here is a list of DOs and DON'Ts for the lab assignments:

  • DO work on the activity portion with a group and share all work from the activity with the rest of your group.
  • DO include all of the group work when you submit the lab assignment.
  • DO include the names of all group members on the group work you submit.
  • DO discuss write-up questions with other students as much as you want. Their opinions are valuable and may help you better understand the activity.
  • DON'T copy work from another group, the internet, an LLM, or other sources. All submitted group work must consist of material I gave you and material that you and your group created during the activity.
  • DON'T copy answers to the written portion of the assignment, have somebody else write them for you, or write them collectively with your group. The written portion is to be completed on your own, and it should only consist of your answers to these questions.
  • DON'T use ChatGPT or another LLM to write or fix your answers to the written portion of the assignment. The write-up should only consist of your own writing. I'm only interested in hearing your thoughts about the activity, and I don't care if you make mistakes as long as you can express your thoughts coherently in English.

I hope the way I designed this class removes most incentives to cheat. However, if I do have to handle an AI violation then I will use the standard penalties listed below. They are different from the penalties listed in the main AI policy for the lecture.

  • I still consider it non-negotiable that you receive no credit for an assignment on which you cheat and are not allowed to redo the assignment.
  • I will count cheating as two incomplete lab assignments rather than one when calculating your final grade. This means cheating on one lab and completing all the others will result in a C (whereas just skipping that assignment would give you a B), and cheating on two or more assignments will result in an F.
